⚡ What is HLD?

High Level Design focuses on the architecture of a large-scale distributed system — scalability, availability, reliability, and trade-offs. Think: "Design Twitter", "Design Netflix", "Design URL Shortener".

The HLD Interview Framework
Follow this structure every time
STEP 1 — 5 min
Clarify Requirements
Functional requirements (what it does), Non-functional (scale, latency, availability, consistency). Ask: DAU, read/write ratio, data size, SLA.
STEP 2 — 5 min
Capacity Estimation
Back-of-envelope: QPS, storage, bandwidth. E.g. 100M DAU × 10 requests/day = ~11,500 QPS. Show your math — interviewers love this.
STEP 3 — 5 min
High-Level Architecture
Draw: Client → API Gateway → Services → DB/Cache. Identify core components. Define API endpoints briefly.
STEP 4 — 15 min
Deep Dive Components
Pick 2–3 tricky parts (DB schema, caching layer, message queue, consistency). Go deep. Discuss trade-offs proactively.
STEP 5 — 5 min
Bottlenecks & Scale
Where does your system break at 10×? Discuss sharding, horizontal scaling, failover, monitoring, alerting.

⚡ What is LLD?

Low Level Design focuses on class design, OOP principles, and design patterns. Think: "Design a Parking Lot", "Design Chess Game", "Design an Elevator System". It's about code structure, not infrastructure.

🧱
OOP Principles — Nail These First
The foundation of every LLD answer
SOLID Principles Must Know
  • Single Responsibility — one class, one job
  • Open/Closed — open to extend, closed to modify
  • Liskov Substitution — subclass is substitutable
  • Interface Segregation — small, focused interfaces
  • Dependency Inversion — depend on abstractions
OOP Pillars Must Know
  • Encapsulation — hide internal state
  • Abstraction — expose only what's needed
  • Inheritance — reuse via is-a relationships
  • Polymorphism — one interface, many forms
🎨
Design Patterns to Know
GoF patterns that actually appear in interviews
Creational High Frequency
  • Singleton — Logger, Config, DB Connection pool
  • Factory / Abstract Factory — Payment processors, Notification types
  • Builder — Complex objects like Query builders, HTTP requests
Structural High Frequency
  • Adapter — Wrap incompatible interfaces
  • Decorator — Pizza toppings, I/O streams, middleware
  • Facade — Simplify complex subsystems
  • Composite — File system tree, UI components
Behavioral High Frequency
  • Observer — Event systems, notifications, pub/sub
  • Strategy — Sorting algorithms, payment methods
  • Command — Undo/redo, task queues
  • State — Vending machine, traffic lights, order status
  • Template Method — Game loop, data processing pipelines

LLD Interview Framework
The right approach in the interview room
STEP 1 — 3 min
Gather Requirements
Functional: what actors exist? What actions can they perform? Non-functional: scale, concurrency, extensibility needs.
STEP 2 — 5 min
Identify Entities & Relationships
List nouns = potential classes. List verbs = potential methods. Define is-a (inheritance) vs has-a (composition) relationships.
STEP 3 — 5 min
Design Class Diagram
Draw classes with attributes, methods, access modifiers. Show interfaces/abstract classes. Show associations and multiplicities.
STEP 4 — 15 min
Code the Core Classes
Write actual code for 2–3 key classes. Apply design patterns where natural. Show error handling and edge cases.
STEP 5 — 5 min
Walk Through a Use Case
Trace a complete user action through your design. Discuss extensibility: "If we add feature X, we only need to change Y."

Problem-Solving Protocol
Do this for EVERY problem
BEFORE CODING
Read → Examples → Edge Cases
Read the problem twice. Trace through 2 examples by hand. Ask: what if input is empty? Negative numbers? Duplicates?
THINK ALOUD
Brute Force → Optimize
State the brute force O(n²) first. Then identify the bottleneck. Ask: can I use a hashmap? Sorting? Sliding window? Binary search?
CODING
Clean Code, Talk as You Type
Use meaningful variable names. Write helper functions. Comment non-obvious logic. Don't go silent.
AFTER CODING
Trace → Complexity → Test
Manually trace your code with an example. State time and space complexity. Test edge cases out loud.

Arrays

Subarrays ▾
Sliding Window
Use when the problem involves a contiguous subarray with a constraint on sum, length, or character count. Fixed-size: move window by 1 each step. Variable-size: shrink left when constraint is violated.
O(n) time O(1)–O(k) space
  1. Initialize left pointer and window state (sum, count map, etc.)
  2. Expand right pointer one element at a time
  3. If constraint violated, shrink from left until valid
  4. Update result at each valid window state
Max Sum Subarray of Size K Longest Substring Without Repeats Minimum Window Substring Fruits Into Baskets
Two Pointers
Use on sorted arrays for pairs/triplets summing to a target. Place l=0, r=n-1, move based on sum comparison.
O(n) time O(1) space
  1. Sort array if not already sorted
  2. Place left at start, right at end
  3. If sum < target, move left right; if sum > target, move right left
  4. For 3Sum, fix one element and run two pointers on the rest
Two Sum (Sorted) Three Sum Container With Most Water Trapping Rain Water
Kadane's Algorithm
Classic max subarray sum. curr = max(x, curr+x) at each element; track global best. For max product, track both min and max.
O(n) time O(1) space
  1. Initialize currentMax = arr[0], globalMax = arr[0]
  2. For each element: currentMax = max(element, currentMax + element)
  3. Update globalMax = max(globalMax, currentMax)
  4. For product variant: track currentMin too (negative × negative = positive)
Maximum Subarray (LC 53) Maximum Product Subarray Best Time to Buy/Sell Stock Circular Subarray Max Sum
Prefix Sum + Hash Map
Build running sums; if prefix[i] - prefix[j] = k, subarray [j+1..i] has sum k. Store prefix counts in a hash map for O(1) lookup.
O(n) time O(n) space
  1. Initialize prefix = 0 and hashmap {0: 1}
  2. For each element, add to prefix sum
  3. Check if (prefix - k) exists in map → add its count to result
  4. Store current prefix count in map
Subarray Sum Equals K (LC 560) Longest Subarray With 0 Sum Binary Array Equal 0s and 1s Count Subarrays With Even Sum
Divide & Conquer
Split at midpoint, solve each half recursively, compute best cross-midpoint subarray. Useful for counting inversions or range-sum queries.
O(n log n) time O(n) space
  1. Split array at midpoint into left and right halves
  2. Recursively solve for each half
  3. Compute best subarray crossing the midpoint (expand from mid both ways)
  4. Return max of left, right, and crossing results
Maximum Subarray (D&C) Count Inversions (LC 315) Reverse Pairs Count of Range Sum
Dynamic Programming
dp[i] = best subarray ending at index i. Kadane's is the 1D DP case. Generalizes to 2D for substrings.
O(n)–O(n²) time O(n) or O(1) space
  1. Define dp[i] as the optimal value for subarray ending at index i
  2. Write recurrence: dp[i] = f(dp[i-1], arr[i])
  3. Handle base case dp[0] = arr[0]
  4. Track global optimum across all dp values

Sorting ▾
Quick Sort
Use when average O(n log n) is acceptable and in-place sorting is preferred. Unstable. Best general-purpose sort; avoid on nearly-sorted data.
O(n log n) avg, O(n²) worst O(log n) space
  1. Pick a pivot (median-of-three or random)
  2. Partition: elements < pivot go left, > pivot go right
  3. Recursively sort left and right partitions
Sort an Array Sort Colors (Dutch National Flag) Kth Largest Element
Merge Sort
Use when stable sort is needed or sorting linked lists. Guaranteed O(n log n) worst case. Preferred for external sorting.
O(n log n) time O(n) space
  1. Divide array in half recursively until single elements
  2. Merge sorted halves by comparing elements one-by-one
  3. Copy merged result back
Sort an Array Merge Sorted Array Count Smaller After Self
Counting Sort
Use when values are integers in a known small range [0, k]. O(n) when k is small. Not comparison-based.
O(n + k) time O(k) space
  1. Count frequency of each value in a counts array
  2. Compute prefix sums of counts for stable placement
  3. Place each element at its computed position
Sort Colors Maximum Gap H-Index
Cyclic Sort
Use when array contains numbers in range [1, n] or [0, n]. Each number has exactly one correct position. O(n) swaps to place all correctly.
O(n) time O(1) space
  1. For each index i, while arr[i] is not at its correct position, swap it there
  2. After sorting, scan for mismatches to find missing/duplicate numbers
  3. Elements already in place are skipped in O(1)
Missing Number Find All Duplicates First Missing Positive Find the Duplicate Number
Searching ▾
Standard Binary Search
Use when array is sorted and you need to find a target or insertion point. Halves the search space each step.
O(log n) time O(1) space
  1. Set lo = 0, hi = n - 1
  2. Compute mid = lo + (hi - lo) / 2
  3. If arr[mid] == target, return; if less, lo = mid + 1; else hi = mid - 1
Binary Search (LC 704) Search Insert Position Guess Number Higher or Lower
Left / Right Boundary Search
Use when finding first or last occurrence of target in sorted array with duplicates. Modify binary search to continue after finding target.
O(log n) time O(1) space
  1. For left boundary: when arr[mid] == target, set hi = mid - 1 and record
  2. For right boundary: when arr[mid] == target, set lo = mid + 1 and record
  3. Return recorded position (or -1 if never found)
First and Last Position (LC 34) Count Negatives in Sorted Matrix Search a 2D Matrix
Rotated Array Search
Use when a sorted array has been rotated and you need to find a target or the minimum element.
O(log n) time O(1) space
  1. Compute mid; determine which half [lo..mid] or [mid..hi] is sorted
  2. Check if target falls within the sorted half
  3. If yes, search that half; otherwise search the other half
Search in Rotated Array (LC 33) Find Min in Rotated Array Rotated Array II (with dups)
Other Patterns ▾
Frequency Map / Hash Counting
Use when counting occurrences, detecting duplicates, or finding mode/majority elements. Hash map trades space for O(1) lookups.
O(n) time O(n) space
  1. Iterate array, count each element in a hash map
  2. Query map for the required property (most frequent, pairs, etc.)
  3. For constant-space alternative, consider Boyer-Moore voting

In-place Manipulation
Use when modifying an array without extra space. Leverage indices as implicit storage, or overwrite from one end.
O(n) time O(1) space
  1. Use a write pointer to track where next valid element goes
  2. Iterate with read pointer, copy valid elements to write position
  3. For rotations, use reverse trick: reverse whole, reverse parts

Variable-Size String Window
Use when finding substrings with character-based constraints (unique chars, at most k distinct, anagram match). Maintain char frequency map in window.
O(n) time O(k) space

Converging Pointers
Use for palindrome checks, string reversals, or comparing characters from both ends. Skip non-alphanumeric for "valid palindrome" variants.
O(n) time O(1) space
  1. Place left at index 0, right at end
  2. Compare characters, move inward
  3. For palindrome II, try skipping left or right on mismatch

Hashing / Anagram Detection ▾
Character Frequency Matching
Use to detect anagrams, group words by character signature, or find pattern occurrences. Fixed 26-char array gives O(1) space for lowercase.
O(n) time O(1) space (26 chars)
  1. Build frequency array/map of pattern
  2. For grouping: sort chars or use frequency signature as key
  3. For sliding: add right char, remove left char, compare with pattern

KMP / Rabin-Karp Pattern Matching ▾
KMP (Knuth-Morris-Pratt)
Use for exact pattern matching in O(n+m) without backtracking. Builds a failure function to skip redundant comparisons.
O(n + m) time O(m) space
  1. Build prefix/failure array: lps[i] = length of longest proper prefix that is also suffix of pattern[0..i]
  2. Scan text with two pointers; on mismatch, jump pattern pointer using lps array
  3. On full match, record position and continue using lps
Implement strStr Repeated Substring Pattern Shortest Palindrome
Rabin-Karp (Rolling Hash)
Use for multi-pattern matching or when you need to compare many substrings quickly. Rolling hash avoids recomputing from scratch.
O(n) average O(1) space
  1. Compute hash of pattern
  2. Roll hash across text: add new char, remove old char in O(1)
  3. On hash match, verify characters to avoid false positives

Trie for Prefix Lookups
Use when doing many prefix-based lookups over a dictionary of words. Each node stores children and end-of-word flag.
O(L) per operation O(N × L) total
  1. Build trie by inserting words char-by-char
  2. Search by following child pointers for each character
  3. For prefix queries, check node exists without requiring isEnd

Monotonic Stack ▾
Next Greater / Smaller Element
Use when finding the next greater or smaller element for each position. Stack maintains decreasing (or increasing) order of unresolved elements.
O(n) time O(n) space
  1. Iterate array left-to-right (or right-to-left)
  2. Pop stack while top violates monotonic property vs current
  3. For each popped element, current is its answer
  4. Push current index/value onto stack
Next Greater Element I & II Daily Temperatures Largest Rectangle in Histogram Trapping Rain Water
Monotonic Deque ▾
Sliding Window Maximum / Minimum
Use when finding max or min in every window of size k. Deque stores indices in monotonically decreasing (for max) order.
O(n) time O(k) space
  1. Maintain deque of indices; front is always the window max
  2. Remove indices from back that are smaller than current element
  3. Remove front if it falls outside the window
  4. Front of deque is the answer for the current window
Sliding Window Maximum (LC 239) Shortest Subarray With Sum ≥ K Longest Subarray With Abs Diff
Stack for Expression Evaluation ▾
Parentheses & Expression Parsing
Use for matching brackets, evaluating postfix/infix expressions, or decoding nested strings. Push on open, process on close.
O(n) time O(n) space
  1. Push opening brackets or operands onto stack
  2. On closing bracket or operator, pop and evaluate/match
  3. For nested patterns (e.g., "3[a2[c]]"), push context and recurse
  4. Final result is on top of stack (or is the remaining string)

Queue / BFS Patterns ▾
BFS with Queue
Use for level-order tree traversal or shortest path in unweighted graphs. Processes nodes layer by layer.
O(V + E) time O(V) space
  1. Enqueue start node(s), mark visited
  2. Process all nodes at current level (loop over queue size)
  3. For each node, enqueue unvisited neighbors
  4. Increment level/distance counter between layers

Priority Queue / Heap ▾
Heap for Top-K / Streaming
Use when maintaining K best elements from a stream, or merging K sorted sequences. Min-heap of size k for "k largest" problems.
O(n log k) time O(k) space
  1. Insert elements into a min-heap (for k largest) or max-heap (for k smallest)
  2. When heap exceeds size k, pop the root
  3. After processing all elements, heap contains the top-k

Tree Traversals ▾
Inorder / Preorder / Postorder
Fundamental tree traversals. Inorder on BST gives sorted order. Preorder for serialization. Postorder for bottom-up computation. Iterative uses explicit stack.
O(n) time O(h) space
  1. Recursive: base case null, recurse left/right with visit at correct position
  2. Iterative inorder: push all lefts, pop and visit, then go right
  3. Iterative preorder: visit node, push right then left child
  4. Iterative postorder: use two-stack method or reverse modified preorder
Inorder Traversal Preorder Traversal Postorder Traversal Construct from Preorder+Inorder
BFS (Level-Order / Shortest Path) ▾
Level-Order / Shortest Path BFS
Use for level-by-level processing in trees or shortest path in unweighted graphs. Queue processes nodes in order of distance from source.
O(V + E) time O(V) space
  1. Enqueue root/source, mark visited
  2. Process level: loop over current queue size, add children/neighbors
  3. Track level number for depth, zigzag, or bottom-up order
  4. For shortest path, distance = level when target is first found
Level Order Traversal Zigzag Level Order Minimum Depth of Tree Shortest Path in Binary Matrix
DFS (Path Finding / Components / Cycles) ▾
DFS Traversal
Use for exploring all paths, detecting cycles, counting connected components, or any problem needing full graph/tree exploration.
O(V + E) time O(V) space
  1. Visit node, mark as visited (or "in-progress" for cycle detection)
  2. Recurse on all unvisited neighbors
  3. For cycle detection in directed graphs, use 3-state coloring (white/gray/black)
  4. Backtrack by restoring state if needed (e.g., path-finding)

Binary Search Tree ▾
BST Operations
Use BST property (left < root < right) for O(log n) search, insert, delete. Inorder traversal yields sorted sequence.
O(h) per operation O(n) for tree
  1. Search: compare target with node, go left if smaller, right if larger
  2. Insert: search for position, create node at null spot
  3. Delete: 3 cases — leaf (remove), one child (replace), two children (swap with inorder successor then delete)
  4. Validate: inorder must be strictly increasing, or pass min/max bounds recursively
Validate BST Kth Smallest in BST Delete Node in BST Sorted Array to BST
Lowest Common Ancestor ▾
LCA via Recursion
Use when finding the lowest shared ancestor of two nodes. For BST, leverage ordering. For general tree, use post-order DFS.
O(n) time O(h) space
  1. If current node is null or equals p or q, return it
  2. Recurse on left and right subtrees
  3. If both return non-null, current node is the LCA
  4. If only one is non-null, propagate that one up
LCA of Binary Tree (LC 236) LCA of BST (LC 235) LCA of Deepest Leaves
Graph Algorithms ▾
Topological Sort (Kahn's BFS)
Use for dependency ordering in DAGs. If a cycle exists, topological sort is impossible (detects cycles). Also solvable via DFS post-order.
O(V + E) time O(V) space
  1. Compute in-degree for each node
  2. Enqueue all zero-degree nodes
  3. Process queue: for each node, reduce neighbors' in-degrees
  4. If new in-degree = 0, enqueue; if result < V nodes, cycle exists
Course Schedule Course Schedule II Alien Dictionary
Union-Find / Disjoint Set Union
Use for dynamic connectivity queries, counting components, and detecting cycles in undirected graphs. Near O(1) per operation with path compression + union by rank.
O(α(n)) per op O(n) space
  1. Initialize parent[i] = i, rank[i] = 0
  2. Find: follow parent pointers with path compression (point to root)
  3. Union: merge smaller-rank tree under larger-rank root
  4. Connected: check if find(a) == find(b)
Connected Components Redundant Connection Accounts Merge Most Stones Removed
Dijkstra's Algorithm
Use for shortest path in weighted graphs with non-negative edges. Priority queue greedily picks the closest unvisited node.
O((V+E) log V) time O(V) space
  1. Initialize dist[source] = 0, all others = infinity
  2. Push (0, source) into min-priority queue
  3. Pop minimum, skip if already settled, relax all edges
  4. If neighbor's new dist < current dist, update and push
Network Delay Time Path With Minimum Effort Cheapest Flights Within K Stops
Bellman-Ford
Use when graph has negative-weight edges. Slower than Dijkstra but handles negatives. Can detect negative cycles.
O(V × E) time O(V) space
  1. Initialize dist[source] = 0, all others = infinity
  2. Relax all edges V-1 times
  3. On Vth pass, if any edge still relaxes, negative cycle exists
  4. For K-stops constraint, only relax from previous round's distances
Cheapest Flights (K stops) Negative Cycle Detection
Grid Problems ▾
Grid DFS / BFS
Treat grid as a graph: each cell is a node with 4 (or 8) neighbors. Use DFS/BFS for connected components, flood fill, or shortest distance.
O(m × n) time O(m × n) space
  1. Iterate through grid, on unvisited target cell start DFS/BFS
  2. Mark cells visited (modify grid or use visited set)
  3. Explore 4 directions, check bounds and visit conditions
  4. Count components, compute distances, or transform cells

Fast & Slow Pointers ▾
Floyd's Cycle Detection / Middle Node
Use when detecting cycles, finding the cycle start, or locating the middle element. Slow moves 1 step, fast moves 2 steps.
O(n) time O(1) space
  1. Initialize slow = head, fast = head
  2. Move slow by 1, fast by 2 each step
  3. If they meet, cycle exists; to find start, reset one to head and move both by 1
  4. For middle: when fast reaches end, slow is at the middle

In-place Reversal ▾
Reverse Linked List
Use when reversing all or part of a linked list. Track prev, curr, next pointers and reassign links.
O(n) time O(1) space
  1. Initialize prev = null, curr = head
  2. Save next = curr.next
  3. Point curr.next = prev
  4. Advance: prev = curr, curr = next; repeat until curr is null

Merge Techniques ▾
Merge Sorted Lists
Use when combining two or more sorted linked lists into one sorted result. Two lists: linear merge. K lists: min-heap.
O(n+m) / O(N log k) O(1) / O(k) space
